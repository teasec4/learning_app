# Создание контента

## Поток создания карточек

Создание карточек происходит **внутри конкретной колоды** через `DeckDetailPage`.

### Bottom sheet выбора метода

На `DeckDetailPage` есть FAB `+`. При нажатии открывается bottom sheet с методами:

| Метод | Статус | Описание |
|-------|--------|----------|
| **Manual** | ✅ Active | Открывает `AddWordPage` — форма: слово (иероглиф), пиньинь, перевод, контекст, заметка, источник |
| **AI generate** | 🔜 Coming soon | Генерация карточек по теме через внешний API |
| **Import from text** | 🔜 Coming soon | Разбор текста на слова, добавление как карточек |

### Ручной ввод (AddWordPage)

Форма с полями:
- **Word (character)** — обязательное, китайский иероглиф/слово
- **Pinyin** — транскрипция
- **Translation** — обязательное, перевод
- **Sentence context** — предложение-пример
- **Notes** — заметки пользователя
- **Source** — откуда взято слово

Создание и редактирование — общая страница `AddWordPage` (edit-mode при передаче `editCard`).

### Создание колоды

На `DecksPage` FAB "New deck" → диалог (название + опциональное описание).

## AI-генерация (запланировано)

- Пользователь вводит тему ("еда", "путешествия")
- AI генерирует 10-20 карточек: слово + пиньинь + перевод + пример
- AI-сервис — отдельный API (Flutter → Go backend → LLM)

## Импорт из текста (запланировано)

- Пользователь вставляет текст (статья, диалог)
- Приложение разбивает на слова, предлагает добавить как карточки
- В будущем: CSV/Anki импорт
